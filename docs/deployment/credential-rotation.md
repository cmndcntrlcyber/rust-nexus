# Credential Rotation

Procedures for rotating secrets used by the Nexus deployment. Follow the
recommended schedule unless incident-driven rotation is required.

| Credential | Recommended cycle | Procedure |
|---|---|---|
| mTLS certificates | 90 days | [Cert rotation](#cert-rotation) |
| Cloudflare API token | 180 days | [API token rotation](#cloudflare-api-token-rotation) |
| NodeIdentity keys | On compromise only | [Identity re-key](#nodeidentity-re-key) |

---

## Cloudflare API Token Rotation

1. **Generate a new token** in the Cloudflare dashboard
   (https://dash.cloudflare.com/profile/api-tokens). The token needs:
   - `Zone > DNS > Edit`
   - `Zone > SSL and Certificates > Edit` (if using CF Origin CA / Client
     Certs via `nexus-server pki init --cf-token`)
2. **Update `nexus.toml`** on the C2 host:
   ```toml
   [cloudflare]
   api_token = "<new-token>"
   ```
3. **Restart nexus-server** to pick up the new token:
   ```bash
   sudo systemctl restart nexus-server
   ```
4. **Revoke the old token** in the Cloudflare dashboard.
5. Verify DNS operations work:
   ```bash
   curl -sH "Authorization: Bearer <new-token>" \
     "https://api.cloudflare.com/client/v4/user/tokens/verify" | jq .
   ```

## Cert Rotation

mTLS certificates are generated by `scripts/gen-certs-prod.sh` and
distributed to server, agents, and operators.

1. Re-run `gen-certs-prod.sh` with `--out ./certs/prod`:
   ```bash
   ./scripts/gen-certs-prod.sh \
     --domain c2.example.com \
     --ip 203.0.113.10 \
     --agents 5 \
     --out ./certs/prod
   ```
   Add `--cf-token` / `--cf-zone-id` to use Cloudflare-signed certs.
2. Install new server certs on the C2 host:
   ```bash
   sudo cp certs/prod/server.crt.pem /etc/nexus/server.crt.pem
   sudo cp certs/prod/server.key.pem /etc/nexus/server.key.pem
   sudo systemctl restart nexus-server
   ```
3. Rebuild agent bundles and redeploy:
   ```bash
   ./scripts/build-agent-bundles.sh --force
   ```
4. Distribute new operator/console certs and relaunch the console with
   the updated `CERT_DIR`.

Agents reconnect automatically within ~60 seconds of a server restart.

## NodeIdentity Re-Key

Each nexus-server instance generates an Ed25519 `NodeIdentity` on first
run (`nexus-server --init-identity`). The identity file lives at
`/var/lib/nexus/identity.bin`.

Re-keying is only required if the private key is compromised:

1. Stop the server:
   ```bash
   sudo systemctl stop nexus-server
   ```
2. Back up and remove the old identity:
   ```bash
   sudo mv /var/lib/nexus/identity.bin /var/lib/nexus/identity.bin.revoked
   ```
3. Generate a new identity:
   ```bash
   sudo -u nexus nexus-server --init-identity
   ```
4. Re-generate all mTLS certs (the CA is derived from the identity):
   follow the [Cert rotation](#cert-rotation) steps above.
5. Start the server and redeploy agent bundles:
   ```bash
   sudo systemctl start nexus-server
   ```

All connected agents will need new client certs — treat this as a full
cert rotation cycle.
