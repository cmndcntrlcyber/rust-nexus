## Stage 2.2 — Wrap secret fields with SecretString

`nexus-infra/src/config.rs:34` defines `pub api_token` in
`CloudflareConfig`. This is the only secret field in config structs.

**Implementation notes:**
- Used `secrecy` 0.8 (not 0.10) — version 0.10 changed `SecretString` to
  `Secret<Box<str>>` which breaks `#[derive(Serialize)]`. Version 0.8 uses
  `Secret<String>` which is compatible.
- The `secrecy` crate intentionally does not implement `Serialize` for
  `Secret<String>` (to prevent accidental leaks). Added a custom
  `serialize_redacted()` function that writes `"[REDACTED]"` for the
  `NexusConfig::to_file()` path. Deserialization reads the real value.

**Files modified:**
- `Cargo.toml` — added `secrecy = { version = "0.8", features = ["serde"] }` to workspace deps
- `nexus-infra/Cargo.toml` — added `secrecy = { workspace = true }`
- `nexus-infra/src/config.rs`:
  - Added `use secrecy::{ExposeSecret, SecretString};`
  - Added `fn serialize_redacted()` custom serializer
  - Field: `pub api_token: String` → `#[serde(serialize_with = "serialize_redacted")] pub api_token: SecretString`
  - Default: `String::new()` → `SecretString::new(String::new())`
  - Validation: `.is_empty()` → `.expose_secret().is_empty()`
  - Test: `"test_token".to_string()` → `SecretString::new("test_token".to_string())`
- `nexus-infra/src/cloudflare.rs`:
  - Added `use secrecy::ExposeSecret;`
  - Line 123: `config.api_token` → `config.api_token.expose_secret()`
- `nexus-infra/tests/acme_smoke.rs`:
  - Line 43: `api_token` → `api_token: api_token.into()`

**Scope note:** The `pki.rs` functions (`call_cf_origin_ca_api`,
`call_cf_client_cert_api`) take `api_token: &str` from CLI arguments in
`nexus-server.rs`, not from `CloudflareConfig`. They do not need changes.
