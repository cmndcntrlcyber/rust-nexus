## Stage 5.2 — Replace unmaintained acme-lib

Replaced `acme-lib` 0.8 (unmaintained since 2021, HIGH risk) with
`instant-acme` 0.8 (async, pure-Rust, actively maintained).

**Files modified:**
- `Cargo.toml` — replaced `acme-lib = "0.8"` with `instant-acme = "0.8"`
- `nexus-infra/Cargo.toml` — replaced `acme-lib = { workspace = true }` +
  `openssl = { version = "0.10", features = ["vendored"] }` with
  `instant-acme = { workspace = true }`
- `nexus-infra/src/letsencrypt.rs` — full rewrite of ACME flow:
  - Removed synchronous `acme-lib` + `spawn_blocking` bridge
  - Replaced with native async `instant-acme` flow
  - Same public API: `CertificateManager`, `CertificateInfo`,
    `ChallengeInfo`, `RenewalStatus`
  - Uses `Account::builder()?.create().await` pattern
  - Uses `order.authorizations()` async iterator
  - Uses `order.finalize().await` + `order.poll_certificate().await`
  - Removed openssl dependency entirely (only needed by acme-lib)

**Benefits:**
- No more OpenSSL build dependency (pure-Rust TLS)
- Native async (no `spawn_blocking` bridging)
- Active maintenance and security patches
